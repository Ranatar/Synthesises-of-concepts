# PhiloSynth — Матрицы конфликтности метода и уровня синтеза

## Легенда

| Символ | Значение |
|--------|----------|
| ✓ | Совместимо — директивы не противоречат друг другу |
| ≈ | Мягкое напряжение — разные рамки для одного материала, примиримо при интерпретации |
| ✗ | Умеренный конфликт — несовместимые критерии оценки или процедуры в пределах одного раздела |
| ✗✗ | Жёсткое противоречие — взаимоисключающие директивы об одном и том же объекте |

Сокращения методов: **Д** = Диалектический · **И** = Интегративный · **Дк** = Деконструктивный · **Г** = Герменевтический · **А** = Аналитический · **Тв** = Творческий

## 1. Общая матрица (максимальная степень конфликта по всем разделам)

|  | Д | И | Дк | Г | А | Тв |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **Компаративный** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Трансформативный** | ✓ | ✗✗ | ✓ | ✗ | ✓ | ✓ |
| **Порождающий** | ≈ | ✗✗ | ✓ | ✗ | ✓ | ✓ |

**Устойчивые «безопасные» комбинации для всех уровней:** Деконструктивный, Аналитический, Творческий.

**Безусловно проблемные:** Интегративный + Трансформативный/Порождающий (✗✗ во всех разделах, где оба измерения активны).

## 2. Исполнительное резюме

*Активные карты: `METHOD_SUM` × `LEVEL_SUM_PORTRAITS`*

|  | Д | И | Дк | Г | А | Тв |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **Компаративный** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Трансформативный** | ✓ | ✗✗ | ✓ | ≈ | ✓ | ✓ |
| **Порождающий** | ≈ | ✗✗ | ✓ | ≈ | ✓ | ✓ |

### Конфликты

**Трансформативный × Интегративный** `✗✗`
`LEVEL_SUM_PORTRAITS` добавляет: *«для каждой пары — в чём позиции фундаментально несовместимы?»*
`METHOD_SUM` спрашивает: *«почему именно этот набор философов способен к интеграции, а не только к взаимному отрицанию?»*
Два взаимоисключающих вопроса об одном материале — «Портрет философов».

**Порождающий × Интегративный** `✗✗`
`LEVEL_SUM_PORTRAITS` добавляет: *«что данный философ принципиально НЕ МОЖЕТ увидеть? Какие вопросы его философия вынуждена игнорировать?»*
`METHOD_SUM` спрашивает: *«какие общие основания разделяют традиции?»*
Слепые пятна (то, чего философ не видит) — и общие основания (то, что философы видят совместно) — противоположные рамки для одного раздела.

**Трансформативный × Герменевтический** `≈`
`METHOD_SUM`: *«через какие конкретные точки может произойти слияние горизонтов?»*
`LEVEL_SUM_PORTRAITS`: *«в чём позиции фундаментально несовместимы?»*
Слияние горизонтов предполагает потенциальную совместимость; трансформативный уровень строится на несовместимости. Примиримо, если трактовать «слияние» как результат столкновения.

**Порождающий × Герменевтический** `≈`
`METHOD_SUM`: *«каков горизонт понимания каждого философа и в чём его ограниченность?... через какие точки возможно слияние горизонтов?»*
`LEVEL_SUM_PORTRAITS`: *«фокусируйся на ограничениях — что философ НЕ МОЖЕТ увидеть?»*
Частичное пересечение (ограниченность горизонта ≈ слепое пятно), но «слияние горизонтов» предполагает синтез из существующих перспектив, тогда как порождающий уровень требует выхода за их пределы.

**Порождающий × Диалектический** `≈`
`METHOD_SUM`: *«что выступит тезисом, что — антитезисом? Каков характер снятия?»*
`LEVEL_SUM_PORTRAITS`: *«что данный философ принципиально НЕ МОЖЕТ увидеть?»*
Ограничения философа можно интерпретировать как антитезис, но «снятие» в диалектическом методе подразумевает синтез из позиций философов, а не из проблемы.

## 3. Граф категорий

*Активные карты: `METHOD_GRAPH` × `LEVEL_GRAPH_METHODOLOGY` × `LEVEL_GRAPH_LAST_COL_NAME` × `LEVEL_GRAPH_LAST_COL_SPEC`*

|  | Д | И | Дк | Г | А | Тв |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **Компаративный** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Трансформативный** | ✓ | ✗✗ | ✓ | ≈ | ✓ | ✓ |
| **Порождающий** | ≈ | ✗✗ | ✓ | ✗ | ✓ | ✓ |

### Конфликты

**Трансформативный × Интегративный** `✗✗`
`LEVEL` (методология): *«ЗАПРЕЩЕНО прямое заимствование; категория — НОВОЕ ПОНЯТИЕ, рождённое из НЕВОЗМОЖНОСТИ совместить исходные»*
`METHOD_GRAPH`: *«каждая синтетическая категория обязана иметь явную ссылку на ОБЩЕЕ ОСНОВАНИЕ у ≥ 2 философов»*
Топологическое требование метода — «ядро пересечений» (категории, опирающиеся на большинство традиций) — прямо противоречит уровню: категория, рождённая из невозможности совместить, не может опираться на общее основание.
6-й столбец «Генеалогия» должен содержать «напряжение», но метод ожидает там «общее основание».

**Порождающий × Интегративный** `✗✗`
`LEVEL` (методология): *«категории строятся НЕ из философских систем, а из ПРОБЛЕМЫ; философы — источники ОГРАНИЧЕНИЙ»*
`METHOD_GRAPH`: *«каждая синтетическая категория обязана иметь явную ссылку на общее основание у ≥ 2 философов»*
Категория, выводимая из общего основания традиций, нарушает порождающий принцип.
6-й столбец «Преодолённые ограничения» vs ожидание метода о «разделяемой интуиции».
Требование метода к топологии — «ядро пересечений» — неприменимо: при порождающем уровне нет «пересечений» между традициями как источниками категорий.

**Порождающий × Герменевтический** `✗`
`LEVEL` (методология): *«категории строятся НЕ из философских систем»*
`METHOD_GRAPH`: *«категории организованы через «горизонты понимания»: каждая традиция задаёт свой горизонт, связи = точки слияния горизонтов»*
Организация категорий через горизонты традиций — это и есть организация через философские системы. Требование метода: *«в описании каждой связи укажи, горизонты каких традиций она соединяет»* — теряет смысл, если категории не выводятся из традиций.

**Трансформативный × Герменевтический** `≈`
`LEVEL` (методология): *«категория рождена из НАПРЯЖЕНИЯ, СТОЛКНОВЕНИЯ между философами»*; столбец «Генеалогия»: *«из какого напряжения → механизм разрешения»*
`METHOD_GRAPH`: *«связи между категориями — это точки слияния горизонтов»*; топология: *«движение от частных горизонтов к расширенному общему»*
«Слияние» горизонтов имплицирует совместимость; трансформативный уровень строится на несовместимости как источнике. Примиримо, если толковать «слияние» как результат столкновения.

**Порождающий × Диалектический** `≈`
`LEVEL_GRAPH_LAST_COL_NAME`: «Преодолённые ограничения»
`METHOD_GRAPH`: *«синтетические категории маркируются в столбце происхождения/генеалогии как точки снятия противоречий с явным указанием, какое противоречие снято»*
Один столбец — два ожидаемых содержания: «преодолённые ограничения» (уровень) vs «снятое противоречие» (метод). Модель может интерпретировать ограничение A и ограничение B как тезис и антитезис, что рабочий вариант — но инструкция этого не предписывает явно.

## 4. Глоссарий терминов

*Активные карты: `METHOD_GLOSSARY` × `LEVEL_GLOSSARY_COL` × `LEVEL_GLOSSARY_SEC`*

|  | Д | И | Дк | Г | А | Тв |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **Компаративный** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Трансформативный** | ✓ | ≈ | ✓ | ≈ | ✓ | ✓ |
| **Порождающий** | ≈ | ✗ | ✓ | ✗ | ✓ | ✓ |

### Конфликты

**Порождающий × Интегративный** `✗`
`LEVEL_GLOSSARY_SEC` (generative): *«Термины, преодолевающие ограничения» / «Термины, порождённые проблемой»* — термины, которые по определению не существуют в рамках исходных систем.
`METHOD_GLOSSARY`: *«существуют ли близкие аналоги в других философских традициях? краткое сравнение»*
Вопрос об аналогах логически противоречит подразделу: если термин порождён проблемой и не мог быть сформулирован ни одним философом, поиск аналогов лишён содержания.

**Порождающий × Герменевтический** `✗`
`LEVEL_GLOSSARY_COL` (generative): последний столбец — *«почему ни один из философов не мог сформулировать данный термин»*
`METHOD_GLOSSARY`: *«как понимание термина меняется в зависимости от горизонта интерпретации; что термин означает внутри каждой из исходных традиций»*
Смысл термина «внутри каждой традиции» — требование метода — неприменим к терминам, которые по уровню не существуют внутри ни одной традиции.

**Трансформативный × Интегративный** `≈`
`LEVEL_GLOSSARY_SEC` (transformative): *«Эмерджентные термины — не имеющие прямых предшественников ни у одного из исходных философов»*
`METHOD_GLOSSARY`: *«существуют ли близкие аналоги в других традициях (не только в выбранных)?»*
Метод расширяет поиск за пределы выбранных философов (все традиции), поэтому строгого противоречия нет — но дух раздела (эмерджентность) и дух метода (интеграция через аналоги) расходятся.

**Трансформативный × Герменевтический** `≈`
`METHOD_GLOSSARY`: *«что термин означает внутри каждой традиции — и в расширенном горизонте синтеза»*
`LEVEL_GLOSSARY_COL` (transformative): *«из какого напряжения между философами возникло»*
Расширение горизонтов предполагает непрерывность от исходных значений; трансформативный уровень акцентирует разрыв (термин рождён из невозможности совместить). Примиримо, но создаёт двойственность в постановке задачи.

**Порождающий × Диалектический** `≈`
`METHOD_GLOSSARY`: *«к какому диалектическому моменту (тезис/антитезис/синтез) относится термин?»*
Термины порождающего уровня не происходят из позиций философов, поэтому атрибуция «момента» (чей тезис, чей антитезис) формально не ошибочна, но требует переинтерпретации: ограничение A = антитезис к ограничению B. Рабочий вариант, но неочевидный.

## 5. Корпус тезисов

*Активные карты: `METHOD_THESES` × `LEVEL_THESES_CONTRIB` × `LEVEL_THESES_NOVELTY`*

|  | Д | И | Дк | Г | А | Тв |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **Компаративный** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Трансформативный** | ✓ | ✗✗ | ✓ | ≈ | ✓ | ✓ |
| **Порождающий** | ≈ | ✗✗ | ✓ | ≈ | ✓ | ✓ |

### Конфликты

**Трансформативный × Интегративный** `✗✗`
`LEVEL_THESES_CONTRIB` (transformative): *«какой КОНФЛИКТ породил данный тезис? Не "что внёс", а "какой конфликт породил"»*
`METHOD_THESES`: *«укажи ОБЩЕЕ ОСНОВАНИЕ — принцип/интуицию, разделяемую ≥ 2 философами, из которой тезис вырастает»*
Тезис не может одновременно вырастать из общего основания и рождаться из конфликта. Два обязательных пункта обоснования взаимоисключают друг друга.

**Порождающий × Интегративный** `✗✗`
`LEVEL_THESES_CONTRIB` (generative): *«какое ОГРАНИЧЕНИЕ системы данный тезис преодолевает? Почему тезис НЕВОЗМОЖЕН в рамках его философии?»*
`METHOD_THESES`: *«укажи ОБЩЕЕ ОСНОВАНИЕ — принцип/интуицию, разделяемую ≥ 2 философами»*
Тезис, вырастающий из общего основания традиций, по определению не является невозможным в рамках этих традиций. Прямое противоречие.

**Трансформативный × Герменевтический** `≈`
`METHOD_THESES`: *«как тезис расширяет горизонт понимания по сравнению с каждой традицией? Что становится видимым в расширенном горизонте?»*
`LEVEL_THESES_CONTRIB` (transformative): *«какой конфликт породил данный тезис?»*
«Расширение горизонта» предполагает органическую непрерывность от исходных позиций; «порождение из конфликта» предполагает разрыв. Напряжение в постановке вопроса, но оба могут сосуществовать в одном обосновании.

**Порождающий × Герменевтический** `≈`
`METHOD_THESES`: *«как тезис расширяет горизонт понимания по сравнению с каждой традицией?»*
`LEVEL_THESES_CONTRIB` (generative): *«почему тезис невозможен в рамках философии каждого мыслителя?»*
Невозможность в рамках традиции и расширение горизонта традиции — разные степени разрыва. Примиримо: «невозможен» можно подать как «предельное расширение», но создаётся концептуальная двойственность.

**Порождающий × Диалектический** `≈`
`METHOD_THESES`: *«явный АНТИТЕЗИС — реальная противоположная позиция, опирающаяся на одного из философов»*
`LEVEL_THESES_CONTRIB` (generative): *«ограничение системы данного философа»*
Ограничение как антитезис — рабочая, но нетривиальная интерпретация. Требование метода «опирается на одного из философов» совместимо с уровнем только при трактовке «позиции философа» как его ограничения, а не доктрины.

## 6. Диалог между традициями

*Активные карты: `METHOD_DIALOGUE` × `LEVEL_DIALOGUE_STRUCT` × `LEVEL_DIALOGUE_TABLE` × `LEVEL_DIALOGUE_COMMENT`*

|  | Д | И | Дк | Г | А | Тв |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **Компаративный** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Трансформативный** | ✓ | ✗✗ | ✓ | ✗ | ✓ | ✓ |
| **Порождающий** | ≈ | ✗✗ | ✓ | ✗ | ✓ | ✓ |

### Конфликты

**Трансформативный × Интегративный** `✗✗`
`LEVEL_DIALOGUE_STRUCT` (шаг 3): *«СТОЛКНОВЕНИЕ — обнаружение ФУНДАМЕНТАЛЬНЫХ НЕСОВМЕСТИМОСТЕЙ; в чём именно они НЕ МОГУТ договориться»*
`LEVEL_DIALOGUE_STRUCT` (шаг 4): *«ПОРОЖДЕНИЕ — из НЕВОЗМОЖНОСТИ совместить рождается новое понятие»*
`METHOD_DIALOGUE`: *«участники активно ищут скрытые точки СОГЛАСИЯ; каждый формулирует, что готов принять как верное; тон — КООПЕРАТИВНЫЙ, НЕ ПОЛЕМИЧЕСКИЙ»*
Несовместимые диалоговые процедуры: уровень требует обнаружить, о чём стороны не могут договориться; метод требует, чтобы стороны искали, о чём могут.
`LEVEL_DIALOGUE_TABLE`: «Несовместимые позиции | Суть напряжения | Рождённое понятие» vs ожидаемый таблицей метода акцент на точках согласия.

**Порождающий × Интегративный** `✗✗`
`LEVEL_DIALOGUE_STRUCT` (шаг 2): *«ДЕМОНСТРАЦИЯ ОГРАНИЧЕНИЙ — каждый формулирует, почему проблема НЕРАЗРЕШИМА в рамках его философии»*
`LEVEL_DIALOGUE_STRUCT` (шаг 3): *«совокупность ограничений ВСЁ РАВНО не даёт решения»*
`LEVEL_DIALOGUE_STRUCT` (шаг 4): *«нужен аппарат, не принадлежащий НИ ОДНОМУ из участников»*
`METHOD_DIALOGUE`: *«кооперативный поиск скрытых точек согласия; финал — обнаружение общих оснований»*
Диаметрально противоположные финалы диалога: порождающий уровень — коллективная недостаточность всех систем; интегративный метод — общие основания. Несовместимы как процедурно, так и содержательно.

**Трансформативный × Герменевтический** `✗`
`LEVEL_DIALOGUE_STRUCT` (шаг 3–4): строится вокруг «фундаментальных несовместимостей» и «рождения из невозможности»
`METHOD_DIALOGUE`: *«каждый участник сначала реконструирует горизонт собеседника — точнее и полнее, чем тот сам; финал: что стало видимым в расширенном горизонте; чей горизонт расширился больше»*
Расширение горизонтов — траектория нарастающей совместимости; трансформативный уровень — траектория несовместимости как продуктивного источника. `LEVEL_DIALOGUE_COMMENT` (transformative): оценка несводимости рождённого понятия. `METHOD` производит итоговую таблицу асимметрии горизонтов, уровень — таблицу «Несовместимые позиции | Рождённое понятие». Два разных итоговых артефакта от одного раздела.

**Порождающий × Герменевтический** `✗`
`LEVEL_DIALOGUE_STRUCT` (шаг 5): *«НАБРОСОК НОВОГО ЯЗЫКА — понятия, ВЫХОДЯЩИЕ ЗА ПРЕДЕЛЫ систем участников»*
`METHOD_DIALOGUE`: *«финал фиксирует, что стало видимым в расширенном горизонте; подчеркни асимметрию: чей горизонт расширился больше»*
«Расширенный горизонт» — это всё ещё горизонт из существующих перспектив; «новый язык» порождающего уровня выходит за пределы всех существующих горизонтов. Несовместимые конечные точки диалога.
`LEVEL_DIALOGUE_TABLE`: «Ограничение | Слепое пятно | Новый категориальный запрос» vs герменевтическое «слияние горизонтов».

**Порождающий × Диалектический** `≈`
`LEVEL_DIALOGUE_STRUCT` (шаги 2–4): ограничения → слепые пятна → проблемное поле
`METHOD_DIALOGUE`: *«каждая реплика — тезис, антитезис или попытка снятия; финал = явный момент синтеза, признанного всеми»*
Ограничение философа A можно позиционировать как антитезис к ограничению философа B, а «проблемное поле» — как снятие. Но «синтез, признанный всеми участниками» предполагает позитивный результат из позиций философов, тогда как порождающий уровень настаивает на разрыве с этими позициями.

## 7. Критический анализ

*Активные карты: `METHOD_CRITIQUE` × `LEVEL_CRITIQUE_NOVELTY` × `LEVEL_CRITIQUE_CHECK`*

|  | Д | И | Дк | Г | А | Тв |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| **Компаративный** | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| **Трансформативный** | ✓ | ✗✗ | ✓ | ≈ | ✓ | ✓ |
| **Порождающий** | ≈ | ✗✗ | ✓ | ✗ | ✓ | ✓ |

### Конфликты

**Трансформативный × Интегративный** `✗✗`
`METHOD_CRITIQUE` (integrative): *«для каждого заявленного "общего основания" — насколько оно реально? Таблица асимметрии влияния: Традиция | Доля доминирования %»*
`LEVEL_CRITIQUE_CHECK` (transformative): *«для каждой категории — обосновано ли напряжение? Выяви скрытые заимствования у одного философа, замаскированные под трансформативные»*
Критика оценивает два взаимоисключающих свойства текста: «общие основания» (которых в трансформативном синтезе быть не должно) и «напряжения» (которых в интегративном синтезе быть не должно). Критический аппарат полностью неприменим к тексту противоположного типа.

**Порождающий × Интегративный** `✗✗`
`METHOD_CRITIQUE` (integrative): *«оценка реальности каждого общего основания»; «таблица асимметрии влияния: доля доминирования каждой традиции»*
`LEVEL_CRITIQUE_NOVELTY` (generative): *«порождены ли категории проблемой, а не исходными системами? Шкала 0–10, где 0 = полностью выводим из одного из философов»*
`LEVEL_CRITIQUE_CHECK` (generative): *«убедительно ли обоснована невыводимость из отдельных систем?»*
Метод оценивает качество «общих оснований» традиций. Уровень оценивает, что категории НЕ выводятся из традиций. Критик получает задание оценить критерии, которые в тексте принципиально отсутствуют.

**Порождающий × Герменевтический** `✗`
`METHOD_CRITIQUE` (hermeneutical): *«состоялось ли реальное слияние горизонтов? Один горизонт незаметно подчинил остальные? Является ли итоговый горизонт подлинно новым или это горизонт одного из участников?»*
`LEVEL_CRITIQUE_CHECK` (generative): *«убедительно ли обоснована невыводимость из отдельных систем?»*
*«Слияние горизонтов»* предполагает преемственность от существующих горизонтов; порождающий уровень требует трансценденции всех существующих горизонтов. Критерий оценки «подлинно новый горизонт» (метод) — это всё ещё горизонт, тогда как критерий уровня — полная невыводимость. Несовпадающие системы оценки для одного текста.

**Трансформативный × Герменевтический** `≈`
`METHOD_CRITIQUE` (hermeneutical): *«таблица асимметрии: Философ | Чей горизонт расширился | Чей горизонт стал фоном»*
`LEVEL_CRITIQUE_CHECK` (transformative): *«для каждой категории — из какого напряжения возникла?»*
Обе проверки направлены на подлинность синтеза, но применяют разные критерии: метод ищет асимметрию горизонтов, уровень ищет аутентичность напряжений. Сосуществуют в одном разделе без прямого противоречия, но двоят аналитический фокус.

**Порождающий × Диалектический** `≈`
`METHOD_CRITIQUE` (dialectical): *«таблица: Синтетический элемент | Тезис | Антитезис | Снятие | Оценка диалектичности»*
В порождающем синтезе роль «тезиса» и «антитезиса» могут выполнять ограничения разных философов, а «снятие» — порождённая категория. Таблица формально заполнима, но требует нестандартного переосмысления диалектических ролей.

**Наблюдения для системы рекомендаций:**

- **Трансформативный × Интегративный**: замена метода на Диалектический или Деконструктивный устраняет все конфликты при сохранении уровня; замена уровня на Компаративный устраняет все конфликты при сохранении метода.
- **Порождающий × Интегративный**: аналогично — оба параметра требуют замены; ближайший совместимый метод при сохранении уровня — Деконструктивный или Творческий.
- **Порождающий × Герменевтический**: конфликт умеренный и концентрируется в Графе, Глоссарии и Диалоге; Резюме и Тезисы примиримы. Замена на Деконструктивный снимает все конфликты при сохранении уровня.
- **Трансформативный × Герменевтический**: слабая несовместимость, критична только в Диалоге. Точечная правка `METHOD_DIALOGUE` (hermeneutical) может снять конфликт без замены параметров.
- **Порождающий × Диалектический**: мягкое напряжение во всех разделах — единственная нежёсткая комбинация, где все конфликты ≈. Может работать при явной инструкции интерпретировать «ограничения» как «диалектические моменты».
